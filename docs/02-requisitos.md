# Requisitos Funcionais e Não Funcionais

## Requisitos Funcionais (RF)

1. **Como Participante, quero me inscrever em um evento, para participar.**
   - Critério: formulário enviado com sucesso.

2. **Como Participante, quero pagar a inscrição, para confirmar minha vaga.**
   - Critério: resultado de pagamento exibido.

3. **Como Participante, quero ver meus eventos, para acompanhar minhas inscrições.**
   - Critério: lista de eventos do participante visível.

4. **Como Participante, quero fazer check-in, para registrar presença.**
   - Critério: check-in aceito com código válido.

5. **Como Participante, quero baixar certificado, para comprovar participação.**
   - Critério: certificado disponível após check-in.

6. **Como Organizador, quero criar eventos, para publicar atividades.**
   - Critério: evento aparece em eventos disponíveis.

7. **Como Organizador, quero aprovar inscrições, para controlar participantes.**
   - Critério: inscrição aprovada ou recusada.

8. **Como Admin, quero ver um relatório simples, para monitorar eventos.**
   - Critério: relatório com número de eventos e inscrições.

## Requisitos Não Funcionais (RNF)

1. **Segurança**: bloquear conta após 5 tentativas de login falhas.
2. **Usabilidade**: interface clara e fácil de usar.
3. **Disponibilidade**: página carregando rápido em navegador.
4. **Compatibilidade**: funcionar em Chrome, Firefox e Edge.
5. **Manutenibilidade**: código simples e organizado.
6. **Logs**: registrar ações de login e de admin.
7. **Privacidade**: não mostrar dados de outros usuários.
8. **Erro claro**: mostrar mensagem simples se algo falhar.
